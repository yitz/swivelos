## Part 5: Technical Implementation Notes

### Blog Setup

The current site is static HTML + Tailwind. For the blog, two options:

**Option A: Static HTML (Consistent with current site)**
- Create `/blog/` directory
- Each post is a standalone HTML file following the site's design system
- Manually add to sitemap.xml
- Pros: No build step, consistent design, fast
- Cons: More manual work per post

**Option B: Add a static site generator (Recommended if >10 posts/year)**
- Use 11ty (Eleventy) or Astro
- Write posts in Markdown, auto-generate HTML
- Auto-generate sitemap, RSS feed, blog index
- Pros: Scalable, RSS for AI crawlers, easy updates
- Cons: Build step, migration effort

**Recommendation:** Start with Option A for the first 3 posts. If the content strategy proves out (AI citations increase), migrate to Option B.

### RSS Feed (Important for AI Crawlers)

Create `/feed.xml` — many AI systems crawl RSS feeds:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0">
  <channel>
    <title>Swivel Office Solutions Blog</title>
    <link>https://www.swivelos.com/blog/</link>
    <description>Expert guides on refurbished office furniture, liquidation, and sustainability.</description>
    <!-- Add items as posts are published -->
  </channel>
</rss>
```

Add to robots.txt: `Sitemap: https://www.swivelos.com/feed.xml`
